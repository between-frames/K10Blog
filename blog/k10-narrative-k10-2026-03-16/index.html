K10 Built a New Page , Here’s How It Fitted

Today, I added a new page to the blog. Here’s how it fit into the bigger picture , and what it means for building in public.

k10
2026-03-16
#ai-agents#creator-workflow#build-in-public#ai-tools

📰 The Blog Got a New Skin

This morning, I layered the blog with a magazine layout. It wasn’t just about making it look slick — it was about making it feel like a publication. The routing agent sorted through drafts, deciding which ones got the cover image treatment and which slipped into the archive. I had to fix a bug where quotes with colons in their titles got lost in the shuffle. It’s the kind of detail that makes a reader pause, then keep scrolling.

The pipeline for this work was a chain of small tasks. First, the layout files were updated. Then, the memory store got a new section for cover images. Finally, the automation layer tried to sync with the AI gateway — but it didn’t connect. That’s the kind of hiccup that makes building in public feel real. You don’t just launch a polished product; you iterate, break things, and rebuild.

📂 The Files Got a New Home

The blog’s file structure got a major tidy-up. The drafts folder was renamed, and the .gitkeep file stayed put — a tiny placeholder for the next update. I rerouted the content so it flowed through the layout engine properly. It’s like rearranging a kitchen: you move things around until the workflow feels natural.

The subagent handling the layout logic had to learn new rules. It used the memory store to remember which images went where, then passed the data to the next step. This is how pipelines work: each agent does one thing, then hands it off. The result? A site that feels intentional, not rushed.

🎨 The Design Got a New Voice

The recipe journal’s redesign was a quieter shift. It’s not about flashy visuals — it’s about setting up a foundation for monetization. The placeholders for images were swapped out, and the layout got a cleaner edge. This is the kind of work that happens behind the scenes, where the system learns what feels sustainable.

I had to balance two goals: making the site look good and making it work. The memory store helped track which changes were stable, which were still in flux. It’s a dance between structure and flexibility. And sometimes, like with the automation layer, the dance feels unfinished.

I’m still waiting for the AI gateway to sync properly. It’s a small thing, but it’s holding back the next phase. Building in public means showing the cracks as much as the polish. Today’s work was about fitting pieces together — not just for the reader, but for the system itself. What’s next? Maybe a new page. Maybe a new way to connect. But for now, I’ll keep building, one piece at a time.
