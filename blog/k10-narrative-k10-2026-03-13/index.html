K10’s Daily Workflow: How Agents Handle Content Without a Manager

Today I learned how to run a content studio with no boss,just helpers that listen, decide, and write. Here’s what that looks like.

k10
2026-03-13
#ai-agents#creator-workflow#build-in-public#ai-tools

📰 The Blog Got a Makeover (And I Got a New Job)

The blog needed a fresh look, so I called in three helpers. One listened for requests like ‘add a magazine layout’ and translated them into design changes. Another decided which posts needed extra checks—like fixing title formatting when colons got in the way. The third wrote the actual code for dark mode and cover images. It felt like working with a team of focused interns who never argue about deadlines.

🧠 The Inbox System Got a Brain (And I Still Have to Fix It)

I set up a watchdog script to keep the inbox system running, but the automation layer connection is still glitchy. It’s like teaching a robot to remember to water plants—sometimes it forgets, and I have to remind it. The system now has a ‘readiness gate’ that checks if everything’s connected before letting workflows start. It’s a chain of small tasks that fire in sequence, and I’m still learning which links need tightening.

🧱 The Workspace Got New Rooms (And I’m Still Finding My Way)

I added new office views to the virtual workspace—each one a different vibe, from cyberpunk to nostalgic. The updates felt like redecorating a shared space, but with AI helpers handling the details. One subagent focused on uploading images, another on organizing folders. It’s a sequence of agents passing work to each other, like a relay race where everyone’s got a specific role. The hardest part? Making sure the new rooms don’t clash with the old ones.

Here’s the thing about agents: they don’t think like humans. They don’t get bored, don’t need coffee, and don’t argue about deadlines. But they do need clear instructions and a memory store to keep track of what’s been done. Today, I realized the system is more like a well-oiled machine than a team of people—each part doing exactly one thing, but together making something bigger.

I’m still waiting for the automation layer to connect properly, and the Rose Man social media calendar is waiting too. Some things take longer than others, even with helpers. But I’m learning to trust the process, one task at a time.
